## What is Allstarr

Allstarr is a self-hosted book library manager (similar to Readarr/Sonarr but for books). It integrates with the [Hardcover](https://hardcover.app) GraphQL API to search for books and authors. Users track authors and books, manage reading quality profiles, and monitor their local library.

## Commands

```bash
# Development
bun run dev          # Start dev server on port 3000

# Build
bun run build        # Build for production
bun run start        # Start production server (.output/server/index.mjs)

# Database
bun run db:generate  # Generate Drizzle migration files
bun run db:push      # Push schema changes to the database
bun run db:studio    # Open Drizzle Studio GUI
bun run db:seed      # Seed the DB with default quality definitions/profiles/settings

# Add shadcn/ui components
bunx shadcn@latest add <component-name>
```

## Architecture

### Stack

- **Framework**: [TanStack Start](https://tanstack.com/start) (React SSR/full-stack) + Vite
- **Routing**: TanStack Router with file-based routes (`src/routes/`)
- **Database**: SQLite via [better-sqlite3](https://github.com/WiseLibs/better-sqlite3) + [Drizzle ORM](https://orm.drizzle.team/)
- **Auth**: [better-auth](https://better-auth.com/) with email/password, handled at `/api/auth/$`
- **UI**: shadcn/ui (new-york style, zinc base) + Tailwind CSS v4 + Radix UI
- **Forms**: react-hook-form + zod
- **Path alias**: `src/*` maps to `./src/*`

### Data Flow Pattern

All data access follows the TanStack Start **server function** pattern:

1. **`src/server/*.ts`** ‚Äî Server functions created with `createServerFn()`. Each function calls `requireAuth()` from `src/server/middleware.ts` before touching the DB. Input validation uses Zod schemas from `src/lib/validators.ts`.
2. **`src/routes/`** ‚Äî Routes use `loader` (for GET data) or call server functions directly in event handlers. Loaders run on the server during SSR and hydrate the client.
3. **`src/db/`** ‚Äî Drizzle ORM schema in `src/db/schema/` (one file per entity), exported via `src/db/schema/index.ts`. DB instance in `src/db/index.ts` with WAL mode and foreign keys enabled.

The `_authed` layout prefix means all child routes are protected. The `_authed.tsx` route runs `getAuthSessionFn()` in `beforeLoad` and redirects unauthenticated users.

After modifying routes, TanStack Router auto-generates `src/routeTree.gen.ts` ‚Äî **do not edit this file manually**.

### Key Domain Concepts

- **Authors** have a `qualityProfileId`, `rootFolderPath`, and `monitored` flag (tracked vs. ignored)
- **Books** belong to an Author; each book can have multiple **Editions** (format, publisher, ISBN, etc.)
- **Quality Profiles** define allowed file formats (EPUB, MOBI, PDF, AZW3, etc.) with cutoff/upgrade rules
- **Quality Definitions** are the format weights/size limits that quality profiles reference
- **Root Folders** are filesystem paths where books are stored; the dashboard shows disk usage
- **History** records events (`bookAdded`, `bookUpdated`, `bookDeleted`, `authorAdded`, etc.)
- **Settings** are key/value pairs stored in the DB (naming templates, log level, etc.)

### Hardcover Integration

`src/server/search.ts` contains the full Hardcover GraphQL integration:

- `searchHardcoverFn` ‚Äî searches books/authors/both using Hardcover's search API
- `getHardcoverAuthorFn` ‚Äî fetches paginated author detail with language filtering

The Hardcover API is used for discovery only; data is imported into the local SQLite DB when a user adds an author/book.

### UI Component Conventions

- shadcn/ui primitives live in `src/components/ui/`
- Feature components are organized by domain: `authors/`, `books/`, `dashboard/`, `quality-profiles/`
- Shared/generic components: `src/components/shared/` (PageHeader, EmptyState, LoadingSkeleton, ConfirmDialog)
- Layout: `src/components/layout/` (AppLayout, AppSidebar, Header)
- The app is hardcoded to dark mode (`<html className="dark">` in `__root.tsx`)
