# Phase 1: Multi-Media Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the existing download formats, profiles, settings, and UI to support a two-axis model (contentType + mediaType), rename audiobook to audio, add video format/profile seeds, and prepare the foundation for TV/movie features in later phases.

**Architecture:** A single database migration (0009) handles all schema changes: renames audiobook->audio, adds contentType/enabled/source/resolution columns, migrates settings keys to new namespaces, converts maxSize=0 to NULL, and seeds video formats + profiles. Code changes update all enum references, validators, server functions, and UI components atomically so the existing book functionality remains intact.

**Tech Stack:** SQLite (Drizzle ORM), TanStack Start (React SSR), Zod validators, shadcn/ui, Tailwind CSS v4

**Spec:** `docs/superpowers/specs/2026-03-23-multi-media-support-design.md`
